Word Processing Grade 12

Grade 12 reinforces all Grade 10 and 11 Word skills and adds: document metadata, signature lines, content controls (drop-down lists, restricted text, date pickers), advanced mail merge (multiple data sources), tracking changes, bookmarks, cross-references, custom page setup, advanced linking/embedding and professional document production.

GRADE 12 APPROACH

Grade 12 exams include ALL Grade 10 and 11 Word skills, so revise those pages too. New Grade 12 content focuses on problem-solving and integration — applying skills in unfamiliar scenarios.

Document Properties & Status

Every Word file quietly stores a set of facts about itself in the background — who wrote it, when, how many words it has, and a "status" you can set yourself. This hidden information is called metadata (literally "data about data"). Exams often ask you to change the document's Status property to something like Final or Draft — this is just a label, similar to writing "DRAFT" in pencil on a paper document, except it's stored as a property instead of typed into the text.

Step-by-step: Set the document Status
  1. Click File → Info. This screen shows the document's properties on the right.
  2. Look for the Status field (you may need to click Show All Properties at the bottom to see it).
  3. Click in the empty space next to Status and type the required word, e.g. Final.
  4. Click back into the document (or press Esc) to save the change.
WHY THIS MATTERS

Metadata properties (Author, Status, Title, Subject) don't appear on the printed page — they're for organising and searching files, e.g. a school office filtering for all "Final" exam papers before printing. Don't confuse this with typing the word "Final" into the document text — the exam wants the actual property changed.

Signature Lines

A signature line is a placeholder that shows where someone should sign a printed document — like the dotted line and "Sign here" you'd see at the bottom of a letter, except Word generates it for you with a name, title and date automatically lined up underneath.

Step-by-step: Insert a signature line
  1. Click where the signature should appear, then go to Insert→Signature Line (in the Text group) → Microsoft Office Signature Line…
  2. A "Signature Setup" box opens. Fill in the Suggested signer (their name), Suggested signer's title (their job, e.g. Headmaster) and, if asked, their e-mail address.
  3. If the question gives you "Instructions to the signer" text, type it into that box exactly as given.
  4. Tick or untick Show sign date in signature line depending on what's required.
  5. Click OK. A signature box appears in your document with an X, the name and title printed underneath.

Content Controls & Form Fields

Sometimes a document needs to behave a bit like an app — restricting what someone can type, or giving them a list to choose from instead of free typing. Word calls these content controls, and you insert them from the Developer tab. (If you don't see Developer in the ribbon: File → Options → Customize Ribbon → tick Developer in the right-hand list.)

ControlWhat it's forWhere to set its options
Plain Text Content ControlA box where someone types text — you can limit how many characters fit (e.g. "Itemkode" restricted to 3 characters)Select the control → Developer → Properties → tick Plain Text → set Maximum length, or use Properties dialog depending on Word version
Drop-Down List Content ControlForces the user to pick one option from a fixed list instead of typing anything (e.g. Grade 8–12)Select the control → Developer → Properties → Add… → type each list entry, click OK after each one
Date Picker Content ControlShows a little calendar so the user can click a date instead of typing it, and forces a consistent date formatSelect the control → Developer → Properties → choose a Display Date format (e.g. dd MMMM yyyy)
Combo Box Content ControlLike a drop-down, but the user may also type their own value if it's not in the listSame as Drop-Down List, via Developer → Properties
READ THE ORDER OF OPTIONS CAREFULLY

For a Drop-Down List, the order you click Add… is the order the items appear in the list — if the question says "in descending order" (e.g. Grade 12 down to Grade 8), add them in that order, or use Modify to move an entry up/down afterwards.

Advanced Mail Merge — Multiple Data Sources

In Grade 12, mail merge data sources extend beyond Word tables and Excel:

Data SourceNotes
Word tableTable in a .docx file (first row = field names)
Excel spreadsheetFirst row = field names; sheet name selected during merge
Access databaseChoose a table or query as the data source
CSV fileComma-separated values; compatible with most applications
Email listUse Mailings → Finish → Send Email Messages to merge directly to emails
Step-by-step: Create personalised letters with Mail Merge
  1. Open (or type) your letter in Word — leave gaps where the personal details (name, address) must go.
  2. Click the Mailings→Start Mail Merge button, then choose Letters.
  3. Click Select Recipients → Use an Existing List… and browse to your data source (an Excel file, Access database or Word table). Click Open.
  4. Click in the letter where a name should go. Click Insert Merge Field and choose the field (e.g. FirstName). A placeholder like «FirstName» appears.
  5. Repeat step 4 for every piece of personal information (surname, address, etc.).
  6. Click Preview Results to check how a real letter looks. Use the arrows to flip through each recipient.
  7. Click Finish & Merge → Edit Individual Documents → All → OK. Word creates one new document containing a personalised letter for every recipient.

Mail Merge for Labels

  1. Mailings → Start Mail Merge → Labels
  2. Select label type (Avery, etc.) and size
  3. Select Recipients → choose data source
  4. Insert Address Block or individual merge fields
  5. Update Labels (applies format to all labels)
  6. Finish & Merge → Print or Edit

Links, Bookmarks and Cross-References

Bookmarks

A bookmark marks a specific location in a document so you can navigate to it or link to it.

  • Insert → Bookmark → type name → Add
  • Navigate: Ctrl+G → Bookmark → select name → Go To
  • Create hyperlink to bookmark: Ctrl+K → This Document → select bookmark

Hyperlinks

  • Link to a website: Ctrl+K → Existing File or Web Page → type URL
  • Link to another file: Ctrl+K → Existing File or Web Page → browse to file
  • Link to email address: Ctrl+K → Email Address
  • Remove hyperlink: right-click → Remove Hyperlink

Cross-References

A cross-reference creates a link to another part of the same document (e.g. "See Figure 3 on page 5").

  • References → Cross-reference → choose Reference type (Heading, Figure, Table, Bookmark) and Insert reference to (Page number, Heading text, etc.)
  • When the document changes, select all (Ctrl+A) then press F9 to update all cross-references

Tracking Changes

Tracking changes records all edits made to a document so reviewers can see what was changed.

  • Enable tracking: Review → Track Changes (toggle on/off) — Ctrl+Shift+E
  • Changes appear in a different colour with strikethrough for deletions and underline for insertions
  • Show Markup: Review → Show Markup — choose what types to show
  • Accept change: right-click change → Accept Change (keeps the edit)
  • Reject change: right-click → Reject Change (reverts to original)
  • Accept/Reject All: Review → Accept All Changes / Reject All Changes

Reviewing Tools

Comments
Review → New Comment — adds an annotation balloon without changing the text. Useful for feedback.
Spelling & Grammar
F7 or Review → Spelling & Grammar — checks the entire document
Word Count
Review → Word Count — shows words, characters, paragraphs

Line Breaks and Pagination

Widow and Orphan Control

Widow
Last line of a paragraph appearing alone at the top of a new page (looks bad)
Orphan
First line of a paragraph appearing alone at the bottom of a page (looks bad)

Control these: select paragraph → Home → Paragraph dialog → Line and Page Breaks tab → check "Widow/Orphan control"

Other Line/Page Break Options

  • Keep with next: keeps a heading on the same page as the paragraph that follows
  • Keep lines together: prevents a paragraph from being split across pages
  • Page break before: forces paragraph to start on a new page

Page Setup, Borders & Splitting Tables

These are small, easy-to-lose marks on a document — but they come up constantly because they're quick to test and easy to get wrong if you're rushing.

TaskHow
Change paper size (e.g. A5 instead of A4)Layout→Size → choose A5 from the list, or More Paper Sizes… for a custom size
Set specific margins (e.g. top & bottom = 3 cm)Layout → Margins → Custom Margins → type the values into Top / Bottom / Left / Right
Add a decorative page border ("art border")Design → Page Borders → in the Borders and Shading box, open the Art dropdown and pick a pattern
Apply a border to only the top & bottom of the pageIn the same Page Border dialog, use the preview diagram — click directly on the left/right border lines in the diagram to remove them, leaving only top and bottom
Split a table into two separate tablesClick in the row where the new table should start, then Table Tools → Layout → Split Table. Everything from that row downward becomes a new, separate table — type a heading above it if one is required

Professional Document Production

Grade 12 expects complete, professional documents that combine all skills:

  • Consistent styles (Heading 1, Heading 2, Normal)
  • Automatic Table of Contents
  • Headers/footers with fields (page number, date, author)
  • Properly formatted bibliography using References → Citations
  • Images with captions and cross-references
  • Section breaks for different page orientations/column layouts
  • Mail merge for personalised letters or labels
EXAM TIP — Tracking Changes

In the practical exam you may be asked to: accept/reject specific changes, add a comment, or turn tracking on/off. Know the difference between Accept (keeps the edit) and Reject (removes the edit and restores the original). Changes appear in red by default; each reviewer gets a different colour.
