Spreadsheets Grade 12

Grade 12 adds complex functions (Nested IF, AND/OR, VLOOKUP/HLOOKUP, COUNTIFS/SUMIFS, text functions, date/time functions), freezing multiple rows/columns, custom conditional formatting rules, the Subtotal tool, and advanced charts including stacked graphs and trendlines.

Complex Functions

Nested IF

A plain IF can only sort data into two outcomes — true or false. But what if there are three or more possible results, like turning a mark into "Distinction", "Merit", "Pass" or "Fail"? You solve this by putting a second IF function inside the "false" part of the first one, like a chain of yes/no questions: "Is it 80+? If not, is it 60+? If not, is it 50+? If not, it must be a Fail." That's a nested IF.

Nested IF — Grade classification
=IF(B2>=80, "Distinction",
  IF(B2>=60, "Merit",
    IF(B2>=50, "Pass",
      "Fail")))
COUNTING BRACKETS

Each IF needs a closing bracket. With 3 nested IFs, you need 3 closing brackets ))) at the end. Count: one ) for each IF opened.

AND / OR Functions

AND and OR
<-- AND: all conditions must be TRUE -->
=IF(AND(B2>=50, C2>=50), "Pass", "Fail")

<-- OR: at least one condition must be TRUE -->
=IF(OR(B2>=80, C2>=80), "Has Distinction", "No Distinction")

<-- Combined AND + OR -->
=IF(AND(A2="Active", OR(B2>100, C2>100)), "Yes", "No")

LOOKUP Functions

Imagine a school office with a printed list: ID number on the left, learner's surname on the right. If you know someone's ID and want their surname, you'd run your finger down the ID column until you find a match, then read across to the surname column. VLOOKUP does exactly that — it searches down the first column of a table for a value, then "reads across" to grab a value from another column in the same row.

VLOOKUP — looks up values in a vertical table
=VLOOKUP(lookup_value, table_array, col_index_num, [range_lookup])

Example: Look up student ID in B2, find their mark in column 3 of the lookup table
=VLOOKUP(B2, $E$2:$G$20, 3, FALSE)

Arguments:
  lookup_value = the value to search for (B2)
  table_array  = the range to search in ($E$2:$G$20) — always ABSOLUTE!
  col_index    = which column in the table to return (3 = 3rd column)
  range_lookup = FALSE = exact match; TRUE = approximate match

Error: if not found, returns #N/A
HLOOKUP — looks up values in a horizontal table
=HLOOKUP(lookup_value, table_array, row_index_num, [range_lookup])

Same as VLOOKUP but searches across rows instead of down columns
Step-by-step: Build a VLOOKUP to find a learner's mark
  1. Click the empty cell where you want the answer to appear.
  2. Type =VLOOKUP( to start the function.
  3. Lookup value: click the cell that holds the thing you are searching for (e.g. the learner ID in B2), then type a comma ,
  4. Table array: highlight the whole lookup table (all columns), then press F4 once. This adds dollar signs (e.g. $E$2:$G$20) so the table will not shift when you copy the formula. Type a comma ,
  5. Column number: count which column of that table holds the answer, counting from the LEFT starting at 1. Type that number (e.g. 3 for the 3rd column), then a comma ,
  6. Exact match: type FALSE (this means "only give me an exact match").
  7. Type the closing bracket ) and press Enter. Final formula: =VLOOKUP(B2,$E$2:$G$20,3,FALSE)
EXAM TIP — VLOOKUP

Most common VLOOKUP mistakes: (1) not using absolute reference for table_array ($E$2:$G$20) — it shifts when copied; (2) wrong col_index number — count from the LEFT of the table_array, not from column A; (3) using TRUE instead of FALSE for exact match — always use FALSE in CAT unless told otherwise.

COUNTIFS and SUMIFS

These extend COUNTIF/SUMIF to support multiple criteria:

COUNTIFS and SUMIFS
COUNTIFS: count cells meeting MULTIPLE conditions
=COUNTIFS(A2:A20, "Male", B2:B20, ">=50")
<-- counts rows where A = "Male" AND B >= 50

SUMIFS: sum values meeting MULTIPLE conditions
=SUMIFS(sum_range, criteria_range1, criteria1, criteria_range2, criteria2)
=SUMIFS(C2:C20, A2:A20, "Cape Town", B2:B20, ">100")
<-- sums C where A = "Cape Town" AND B > 100

ROUNDUP and ROUNDDOWN

ROUNDUP / ROUNDDOWN
=ROUNDUP(A1, 0)    <-- always rounds UP to nearest integer (3.1 → 4)
=ROUNDDOWN(A1, 0) <-- always rounds DOWN to nearest integer (3.9 → 3)
=ROUND(A1, 0)      <-- rounds to nearest (3.5 → 4; 3.4 → 3)

Combining Text Functions

LEFT, RIGHT and MID become far more useful once you combine two of them to build a brand-new code out of pieces of other data — for example, a "family code" made from the first 3 letters of a surname plus the last 3 characters of an admission number.

Building a combined code
Surname in A2 = "Adams", Admission number in B2 = "SC2024112"
=LEFT(A2,3) & RIGHT(B2,3)
<-- LEFT(A2,3) grabs "Ada", RIGHT(B2,3) grabs "112", the & glues them together → "Ada112"
READ THE INSTRUCTION IN ORDER

Build these formulas piece by piece exactly in the order the question describes ("first 3 letters of surname, combined with the last 3 characters of admission number") — write down what each piece should produce before joining them with &, so you can spot which half is wrong if the result looks off.

Freezing Multiple Rows or Columns

Freeze Panes keeps certain rows or columns locked in place on screen while you scroll through a large sheet — like keeping the column headings of a printed table visible at the top of every page. Freezing more than one row works the same way as freezing one, you just need to click in the right place first.

Step-by-step: Freeze the top 3 rows
  1. Click on cell A4 — i.e. the cell in column A of the row directly below the rows you want frozen. (To freeze rows 1–3, you click into row 4.)
  2. Go to View→Freeze Panes→Freeze Panes.
  3. Rows 1–3 now stay visible at the top no matter how far down you scroll. A thin grey line marks the frozen boundary.

The same idea works for columns: click in the first cell of the column you want unfrozen (e.g. C1, to freeze columns A and B), then use the same Freeze Panes command.

Conditional Formatting with Custom Rules

Conditional formatting changes how a cell looks automatically, based on what's inside it — so instead of scanning a long column by eye, the spreadsheet highlights the values that matter for you. A common exam task: make every cell containing a specific word (e.g. "No") stand out in bold red.

Step-by-step: Highlight cells containing specific text
  1. Select the range of cells to check (e.g. D2:D20).
  2. Go to Home→Conditional Formatting→Highlight Cells Rules→Equal To… (or Text that Contains… if it isn't an exact match).
  3. Type the word to match, e.g. No.
  4. Click the format dropdown and choose Custom Format… to set both a font style (Bold) and a fill colour (red), then click OK twice.
EXAM TIP — Conditional Formatting vs typing the format by hand

Never select the matching cells manually and apply bold/colour yourself — markers check that the rule is still attached to the cell. If new data is typed in later and it also says "No", it must automatically turn red too. That only happens with a real conditional formatting rule, not manual formatting.

Subtotal Feature

The Subtotal tool automatically inserts summary rows into a list wherever a chosen column's value changes — for example, a sub-total of marks at the end of every grade, without you manually inserting rows or typing SUM formulas yourself. It only works on a sorted, plain list of data (not inside a table object), so sort by the grouping column first.

  • Data → Subtotal → choose column to group by, function (SUM/AVG/COUNT) and column to calculate
  • Creates outline levels — click 1, 2, 3 buttons to expand/collapse groups
  • Remove: Data → Subtotal → Remove All

EXAM TIP — Function Identification

A common Grade 12 exam task: given a scenario, choose the CORRECT function. Key decision points:
• Need to find a value in a table? → VLOOKUP
• Multiple conditions to count? → COUNTIFS
• Multiple conditions to add? → SUMIFS
• Grade classification? → Nested IF
• Extract part of a code? → LEFT / RIGHT / MID
• Calculate how many days since a date? → DAYS or TODAY()
