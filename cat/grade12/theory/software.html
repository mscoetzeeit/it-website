Software & File Management Grade 12

Grade 12 covers the operating system in depth, system utilities, file management (metadata, attributes), performance factors, and comparing web-based vs installed applications.

Role of the Operating System

The OS manages all hardware and software. Its main functions:

Starting the computer
Loads itself into RAM when the computer powers on (bootstrapping/boot process)
User interface
Provides GUI (or command line) for users to interact with the computer
Program management
Loads, runs and closes programs; manages multitasking (multiple programs at once)
Memory management
Allocates RAM to programs; manages virtual memory (using disk space as extra RAM)
File management
Creates, opens, saves, renames, moves and deletes files and folders
Security
User accounts, login, access rights, firewall, anti-spyware/adware
Device management
Communicates with hardware through drivers

Multitasking

Multitasking = running multiple programs apparently simultaneously. The CPU switches between tasks so rapidly it seems parallel. Modern CPUs with multiple cores can actually run tasks in parallel.

Task Manager

Ctrl+Alt+Del → Task Manager. Allows you to: see all running programs and processes, end unresponsive tasks, monitor CPU/RAM/disk/network usage, view startup programs.

File Management — Advanced

File Properties and Metadata

Metadata = data about data — information stored with a file that describes it.

PropertyDescription
File nameThe name of the file (must be unique in the same folder)
File type / extension.docx, .xlsx, .pdf, .jpg — determines which application opens it
File sizeHow much storage space it takes up (bytes, KB, MB, GB)
Date createdWhen the file was first created
Date modifiedWhen the file was last saved/changed
AuthorThe name of who created the file (from Office settings)
TitleA title property set in File → Info → Properties
Location (path)The full folder path where the file is stored

File Attributes

Read-only
File can be opened but NOT edited or deleted; useful for protecting important files from accidental changes
Hidden
File does not appear in standard File Explorer view; used for system files; can be revealed by enabling "Show hidden files"

Set attributes: right-click file → Properties → check/uncheck Read-only or Hidden.

Compressing Files (with Password)

  • Right-click file/folder → Send to → Compressed (zipped) folder (basic Windows compression)
  • Using 7-Zip: right-click → 7-Zip → Add to archive → set password → OK
  • Purpose: reduce file size for email attachment; protect with password for security

Software Maintenance

Patch
Small fix for a specific bug or security vulnerability; does not change overall version number
Update
Larger improvement including multiple patches, performance improvements and minor new features
Upgrade (New version)
Major new release with significant new features (e.g. Office 2019 → Office 2021); usually paid

Factors Influencing Performance

FactorHow it affects performance
RAMMore RAM = more programs running simultaneously without slowing
CPU speed & coresFaster clock speed and more cores = quicker processing
Number of running appsMore apps = more RAM used; CPU splits time between more tasks
CachingFrequently accessed data stored in fast cache = quicker access
Disk optimisationDefragmented HDD or fast SSD = quicker file access
MalwareMalware uses CPU/RAM/network in background = slower system
Startup programsToo many programs starting with Windows = slow boot; disable unnecessary ones
OverheatingPoor ventilation causes CPU throttling = slower performance
EXAM TIP — Patches vs Updates vs Upgrades

These three terms are often confused. Remember: Patch = small specific fix (security hole); Update = bigger collection of fixes + improvements; Upgrade = entirely new version (costs money, big change). An antivirus "definition update" adds new virus signatures — that's an update. Moving from Windows 10 to Windows 11 is an upgrade.
