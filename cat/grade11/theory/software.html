Grade 11 extends software knowledge — covering application software types (collaboration, accessibility), cloud-based apps, software compatibility, installation and file management.

Software Compatibility

COMPATIBILITY ISSUES

Software compatibility refers to whether different software programs can work together correctly. Problems arise when:

  • A file saved in a newer version of Word cannot be opened in an older version (e.g. .docx in Word 2019 may have features not available in Word 2010)
  • Software requires a newer operating system than what is installed
  • A 32-bit program cannot run on a 64-bit OS (or vice versa)
  • Different apps use incompatible file formats

Software Updates

  • Updates (patches): small fixes for bugs or security vulnerabilities; important to install regularly
  • Service pack: a collection of updates, patches and improvements bundled together into one large update
  • Upgrades: new versions with major new features (e.g. Office 2016 → Office 365)
  • Why update? Security (patches vulnerabilities), stability (fixes bugs), new features, compatibility with new hardware
Activation codes & product keys

Much software requires an activation code (product key) during installation. It proves you have a legal licence, limits how many devices may install the software, and helps fight piracy (illegal copying).

Cloud-Based Applications

G Suite (Google Workspace)

  • Google Docs (word processor), Sheets (spreadsheet), Slides (presentation), Forms (surveys)
  • Google Drive (cloud storage), Gmail, Google Meet (video conferencing)
  • Free for individuals; business/education versions available

Microsoft Office 365

  • Web versions of Word, Excel, PowerPoint, Access (online)
  • OneDrive (cloud storage), Teams (collaboration + video), Outlook (email)
  • Subscription-based; always up to date

Advantages of Cloud-Based Apps

  • Access from any device with a browser and internet
  • Real-time collaboration — multiple users edit simultaneously
  • Automatic saving and version history
  • No installation required
  • Always up to date

Disadvantages of Cloud-Based Apps

  • Requires internet connection — cannot work offline
  • Privacy concerns — data stored on third-party servers
  • Fewer features than full desktop versions
  • Depends on provider — if provider closes, data may be at risk
  • Data usage costs (especially on mobile)

System Requirements

Before installing software, check the minimum system requirements — the hardware your computer needs to run the software adequately. Exceeding the minimum (meeting recommended requirements) ensures better performance.

File Management

File types
.docx (Word), .xlsx (Excel), .accdb (Access), .html (web page), .pdf (portable document), .jpg/.png (images), .mp3 (audio), .mp4 (video), .zip (compressed archive)
File properties
Name, size, type, location, date created/modified — right-click file → Properties
File attributes
Read-only (cannot edit), Hidden (not shown in File Explorer by default)
Import
Bring data from an external file into an application (e.g. import CSV into Excel)
Export
Save data from an application in a format usable by another (e.g. export Excel as CSV)
EXAM TIP — Cloud vs Installed Applications

Common exam question: compare web-based (cloud) vs installed (desktop) applications. Key point: cloud apps need Internet; installed apps work offline. Cloud apps auto-update; installed may need manual updates. Cloud stores data on remote servers; installed stores locally. Know at least 2 advantages and 2 disadvantages of each.

Common Exam Mistakes

Windows is application software.
Windows is system software — an operating system.
Freeware means the same as open-source.
Freeware is free but its source code is hidden; open-source shares the code.
Defragmenting frees up disk space.
Defragmenting reorders files for speed; Disk Cleanup frees space.
