Spreadsheets Grade 11

Grade 11 extends on all Grade 10 Excel skills. New topics include absolute references, conditional formatting, advanced functions (IF, COUNTIF, SUMIF, ROUND), charts, sheet manipulation and print options.

Absolute vs Relative Cell References

TypeExampleBehaviour when copied
Relative=B2*C2Adjusts automatically (B3*C3, B4*C4…)
Absolute=B2*$C$1The $ locks the reference — C1 stays fixed
Mixed=$B2*C$1Locks either the column ($B) or row (C$1) only
WHEN TO USE ABSOLUTE REFERENCES

Use $ when a cell should NOT change when the formula is copied. Common example: a VAT rate or tax percentage stored in a single cell that is referenced by many formulas. Press F4 to toggle between relative, absolute and mixed modes.

Example — calculating price with VAT
VAT rate is stored in cell B1 (e.g. 15%)
Formula in C3 to add VAT:
=B3 * (1 + $B$1)    <-- $B$1 stays fixed when copied down column C

Conditional Formatting

Conditional formatting automatically changes the appearance of cells based on their values — no formula in the cell is changed.

Step-by-step: Colour all marks below 50 in red
  1. Highlight the cells that contain the marks: click the first cell, hold the mouse button and drag over all of them, then let go.
  2. Click the Home→Conditional Formatting button (in the Styles group).
  3. In the menu, point to Highlight Cells Rules, then click Less Than…
  4. A small box appears. In the white box, type 50.
  5. Click the dropdown on the right and choose Light Red Fill with Dark Red Text (or "Custom Format…" to pick your own colour).
  6. Click OK. Every mark below 50 is now shaded red automatically. If you change a mark to 60, the red shading disappears on its own.
  • Home → Conditional Formatting → New Rule
  • Highlight Cells Rules: colour cells greater than, less than, between, equal to a value; cells with specific text
  • Data Bars: fills cell with a bar proportional to its value
  • Colour Scales: gradient of colours (e.g. red→yellow→green for low→high)
  • Manage/delete rules: Home → Conditional Formatting → Manage Rules

Advanced Functions

ROUND, ROUNDUP, ROUNDDOWN

FunctionSyntaxExampleResult
ROUND=ROUND(number, digits)=ROUND(3.567, 2)3.57
ROUNDUP=ROUNDUP(number, digits)=ROUNDUP(3.561, 1)3.6
ROUNDDOWN=ROUNDDOWN(number, digits)=ROUNDDOWN(3.569, 1)3.5
ROUNDING ≠ FORMATTING

Formatting a number to 2 decimal places only changes how it looks — the underlying value stays the same. ROUND actually changes the stored value. An exam may ask you to distinguish between these two methods.

SMALL and LARGE

SMALL and LARGE
=SMALL(B2:B20, 1)   <-- smallest value (same as MIN)
=SMALL(B2:B20, 2)   <-- 2nd smallest value
=LARGE(B2:B20, 1)   <-- largest value (same as MAX)
=LARGE(B2:B20, 3)   <-- 3rd largest value

POWER

POWER
=POWER(A1, 3)   <-- A1 to the power of 3 (A1³); same as =A1^3

IF Function

The IF function returns one value if a condition is TRUE and another if FALSE.

IF syntax
=IF(logical_test, value_if_true, value_if_false)

Examples:
=IF(B2>=50, "Pass", "Fail")
=IF(C3<0, "Error", C3)
=IF(A1="Yes", 10, 0)
Step-by-step: Use IF to show "Pass" or "Fail"
  1. Click the cell where the result should appear (e.g. next to the first mark).
  2. Type =IF( to begin.
  3. The test: click the cell with the mark, then type the condition. For a pass mark of 50, type >=50 then a comma , — so far: =IF(B2>=50,
  4. If true: type what to show when the test passes, in quotes: "Pass" then a comma ,
  5. If false: type what to show otherwise, in quotes: "Fail"
  6. Type the closing bracket ) and press Enter. Final: =IF(B2>=50,"Pass","Fail")
  7. To copy it down for everyone: click the cell again, then drag the little square at its bottom-right corner (the fill handle) down the column.

COUNTIF

COUNTIF — count cells meeting a condition
=COUNTIF(range, criteria)

=COUNTIF(B2:B30, ">=50")     <-- count cells ≥ 50
=COUNTIF(C2:C30, "Pass")      <-- count cells containing "Pass"
=COUNTIF(D2:D30, "<>0")      <-- count cells not equal to 0

SUMIF

SUMIF — add cells meeting a condition
=SUMIF(range, criteria, sum_range)

=SUMIF(A2:A20, "Cape Town", B2:B20)
<-- adds values in B2:B20 where A2:A20 = "Cape Town"

=SUMIF(C2:C20, ">100", D2:D20)
<-- adds D values where C > 100

Circular Reference

A circular reference happens when a formula refers back to its own cell, directly or indirectly — like trying to weigh yourself while standing on the scale you're holding. Excel can't calculate it because the answer depends on itself.

Example circular reference (in cell A1)
=SUM(A1:A5)  <-- A1 is trying to include itself in the sum

Excel shows a warning when this happens and displays 0 in the cell until the formula's range is corrected to exclude its own cell.

#NULL! Errors

ErrorCause
#NULL!A formula uses a space (which means "intersection") between two ranges that don't actually overlap, e.g. =SUM(A1:A5 C1:C5)

EXAM TIP — IF vs COUNTIF vs SUMIF

IF → returns a value based on a condition (one cell at a time).
COUNTIF → counts how many cells in a range meet a condition (returns a number).
SUMIF → adds values in a range that meet a condition (returns a total).
